<h2>Ignore rules</h2>
<p>The root <code>.gitignore</code> keeps the repository clean by excluding common generated content before anything is pushed.</p>
<ul>
  <li><code>node_modules/</code> in any nested folder</li>
  <li>build output such as <code>dist/</code>, <code>build/</code>, <code>out/</code>, <code>coverage/</code>, and <code>.next/</code></li>
  <li>logs, temp files, environment files, package manager caches, and editor folders</li>
</ul>
<p>If any generated files were already tracked before the ignore rule was added, they need to be removed from the Git index once so the rule fully takes effect.</p>
